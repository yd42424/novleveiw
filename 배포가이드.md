# 📱 오프라인 소설 뷰어 - GitHub Pages 배포 가이드

## 🎯 완성된 기능

- ✅ **오프라인 열람**: 한번 로드하면 인터넷 없이 읽기 가능
- ✅ **모바일 최적화**: 스와이프로 페이지 넘기기
- ✅ **다크모드**: 라이트/다크/세피아 3가지 테마
- ✅ **진행도 저장**: 어디까지 읽었는지 자동 저장
- ✅ **목차**: 원하는 화로 바로 이동
- ✅ **반응형**: PC, 태블릿, 모바일 모두 지원

## 📁 필요한 파일

```
your-repo/
├── index.html          (방금 만든 뷰어)
└── crawled_novel.json  (크롤링한 소설 데이터)
```

## 🚀 GitHub Pages 배포 방법

### 1단계: GitHub 저장소 만들기

1. GitHub.com 접속 → 로그인
2. 우측 상단 `+` → `New repository` 클릭
3. 저장소 이름 입력 (예: `my-novel-viewer`)
4. **Public** 선택 (Private는 GitHub Pages 유료)
5. `Create repository` 클릭

### 2단계: 파일 업로드

#### 방법 A: 웹에서 직접 업로드

1. 저장소 페이지에서 `uploading an existing file` 클릭
2. `index.html` 파일 드래그 앤 드롭
3. `Commit changes` 클릭
4. 다시 `Add file` → `Upload files`
5. `crawled_novel.json` 파일 업로드
6. `Commit changes` 클릭

#### 방법 B: Git으로 업로드 (추천)

```bash
# 저장소 클론
git clone https://github.com/your-username/my-novel-viewer.git
cd my-novel-viewer

# 파일 복사 (index.html과 crawled_novel.json을 여기에 복사)
# Windows: 탐색기에서 복사/붙여넣기

# Git 추가 및 커밋
git add .
git commit -m "소설 뷰어 추가"
git push
```

### 3단계: GitHub Pages 활성화

1. 저장소 페이지 → `Settings` 탭
2. 왼쪽 메뉴 → `Pages`
3. `Source` → `Deploy from a branch` 선택
4. `Branch` → `main` (또는 `master`) 선택
5. `/ (root)` 선택
6. `Save` 클릭

### 4단계: 배포 확인

- 2-3분 대기
- `https://your-username.github.io/my-novel-viewer/` 접속

## 📱 모바일에서 오프라인으로 보는 방법

### iOS (iPhone/iPad)

1. Safari에서 `https://your-username.github.io/my-novel-viewer/` 접속
2. 아래쪽 공유 버튼(📤) 탭
3. "홈 화면에 추가" 선택
4. 이름 설정 → "추가"
5. 홈 화면에 앱처럼 추가됨
6. **첫 실행 시 모든 데이터 로드됨**
7. 이후 비행기 모드에서도 작동!

### Android

1. Chrome에서 접속
2. 우측 상단 메뉴(⋮)
3. "홈 화면에 추가" 선택
4. 이름 설정 → "추가"
5. 홈 화면에 아이콘 생성
6. 오프라인에서도 작동!

## 🎨 사용 방법

### 기본 조작

- **다음/이전 화**: 화면 좌우 스와이프 또는 하단 버튼
- **목차**: 상단 📚 버튼
- **설정**: 상단 ⚙️ 버튼
- **테마**: 설정에서 라이트/다크/세피아 선택
- **글자 크기**: 설정에서 슬라이더 조절

### PC에서

- **키보드**: ← → 화살표로 페이지 이동
- **스크롤**: 자동으로 헤더 숨김/표시

## 📊 용량 관리

- **JSON 파일**: 약 1-2MB (100화 기준)
- **HTML 파일**: 약 30KB
- **총 용량**: 2MB 이하 (GitHub Pages 무료: 1GB까지)

### 대용량 소설 (1000화 이상)

파일을 여러 개로 나누기:

```javascript
// novel_1-100.json
// novel_101-200.json
// ...
```

## 🆘 문제 해결

### "소설을 불러올 수 없습니다"

- `crawled_novel.json` 파일이 `index.html`과 같은 폴더에 있는지 확인
- 파일 이름이 정확한지 확인 (대소문자 구분)

### GitHub Pages가 안 열림

- 2-3분 대기 (첫 배포는 시간 소요)
- Settings → Pages에서 URL 확인
- 저장소가 Public인지 확인

### 모바일에서 느림

- JSON 파일이 너무 큰 경우 (10MB 이상)
- 파일을 여러 개로 나누기 권장
